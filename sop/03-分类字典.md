# 女皇入朝：职业问题分类字典 V2.2

版本：V2.2
日期：2026-07-19
用途：阶段路由、知识库选择、问题分类、风险识别、检索和评测

## 1. 分类对象

每轮用户输入必须产生以下结构：

```json
{
  "career_phase": "posthire",
  "journey_stage": "DAY_8_30",
  "primary_topic": "workload_priority",
  "secondary_topics": ["manager_communication"],
  "primary_intent": "decide",
  "secondary_intents": ["plan"],
  "risk_level": "normal",
  "content_priority": "core"
}
```

规则：

- 一个且仅一个 `career_phase`；
- 一个且仅一个 `journey_stage`；
- 一个且仅一个 `primary_topic`；
- `secondary_topics` 最多两个；
- 一个且仅一个 `primary_intent`；
- `secondary_intents` 最多两个；
- 一个且仅一个最高风险等级；
- 一个内容优先级。

## 2. 职业阶段 career_phase

| ID | 名称 | 定义 | 知识库 |
| --- | --- | --- | --- |
| `job_seeking` | 未入职找工作 | 尚未接受 Offer，正在探索、准备、投递、面试或选择 Offer | `workplace_job_seeking` |
| `posthire` | 实习/入职后 | 已开始实习、正式工作或转岗进入新团队，且在 12 个月内 | `workplace_newcomer_year_one` |

阶段由用户确认。分类器不得仅凭一句问题覆盖保存阶段。

如果用户明确表示阶段已变化：

1. 返回 `requires_phase_confirmation=true`；
2. 产品要求用户确认；
3. 确认后更新阶段；
4. 使用新阶段重新处理当前问题。

## 3. 旅程阶段 journey_stage

### 3.1 求职阶段

| ID | 定义 |
| --- | --- |
| `DIRECTION` | 探索职业方向、岗位和行业 |
| `PREPARATION` | 盘点能力、补充经历、制作作品和求职材料 |
| `APPLICATION` | 简历定制、渠道、投递和漏斗管理 |
| `INTERVIEW_OFFER` | 面试、谈判、Offer 选择和招聘风险核查 |

### 3.2 实习/入职后阶段

| ID | 定义 |
| --- | --- |
| `DAY_1_7` | 报到、理解职责、工具、流程和关键人 |
| `DAY_8_30` | 建立工作地图、目标和第一次小交付 |
| `DAY_31_60` | 开始独立推进、同步进展和获取反馈 |
| `DAY_61_90` | 形成工作方法、成果记录和阶段复盘 |
| `MONTH_4_6` | 稳定独立交付、跨团队协作和能力补足 |
| `MONTH_7_12` | 稳定贡献、个人标签、年度复盘和下一阶段方向 |

实习用户默认使用同一时间阶段，通过用户画像中的 `employment_type=internship` 调整 SOP 适用条件，不额外制造一套重复阶段。

## 4. 内容优先级 content_priority

| ID | 定义 | 默认展示策略 |
| --- | --- | --- |
| `core` | 入职 0-90 天核心内容 | 主动阶段计划 + 临时检索 |
| `extended` | 入职 4-12 个月主要延伸 | 阶段计划 + 临时检索，模块更少 |
| `secondary` | 未入职求职内容 | 用户进入求职阶段后展示 |
| `event_only` | 晋升、转岗、裁员、离职、副业等事件 | 只在用户主动表达时检索 |
| `routing_only` | 法律、安全、医疗和高风险问题 | 只做边界和帮助路由 |

## 5. 求职主题

| ID | 名称 | 定义 | 默认优先级 |
| --- | --- | --- | --- |
| `career_direction` | 职业方向 | 选择适合的岗位、方向和职业动力 | `secondary` |
| `market_research` | 岗位与行业研究 | 了解岗位内容、行业、公司、城市和市场 | `secondary` |
| `transferable_skills` | 可迁移能力 | 将专业和经历翻译为目标岗位能力 | `secondary` |
| `experience_portfolio` | 经验与作品集 | 用项目、实习、志愿活动和作品建立能力证据 | `secondary` |
| `networking_referral` | 人脉与内推 | 信息访谈、社群、弱连接和内推 | `secondary` |
| `resume_application` | 简历与投递 | 简历定位、定制投递、渠道和漏斗 | `secondary` |
| `interview` | 面试表达 | 自我介绍、经历表达、追问和复盘 | `secondary` |
| `offer_negotiation` | Offer 选择与谈判 | 比较、谈薪、接受或拒绝 Offer | `secondary` |
| `job_seeking_risk` | 招聘风险 | 收费、隐私、歧视、骚扰和岗位异常 | `routing_only` 或 `secondary` |
| `preboarding` | 入职前准备 | 接受 Offer 后的确认和准备 | `secondary`，阶段转换入口 |

## 6. 实习/入职后核心主题

| ID | 名称 | 定义 | 默认优先级 |
| --- | --- | --- | --- |
| `onboarding_landing` | 入职与 Landing | 报到、试用期、业务、流程和关键人 | `core` |
| `organization_stakeholder` | 组织关系与 Stakeholder | 识别职责、评价、资源和影响关系 | `core` / `extended` |
| `workload_priority` | 工作量与优先级 | 任务冲突、容量、排序和减负 | `core` / `extended` |
| `manager_communication` | 向上沟通 | 目标对齐、汇报、反馈和 1:1 | `core` / `extended` |
| `peer_collaboration` | 同事与跨部门协作 | 协作推进、依赖、分工和催进度 | `core` / `extended` |
| `performance_visibility` | 成果与可见度 | 记录贡献、周报、复盘和评价证据 | `core` / `extended` |
| `boundary_conflict` | 边界与冲突 | 拒绝困难、职责争议、抢功和甩锅 | `core` / `extended` |
| `skill_growth` | 能力发展 | 当前交付相关的技能缺口和成长方向 | `core` / `extended` |
| `ai_workflow` | AI 与工作方式 | AI 工具使用、验证责任和协作流程 | `extended` |

## 7. 入职后事件型主题

这些主题不进入默认阶段计划。

| ID | 名称 | 定义 | 默认优先级 |
| --- | --- | --- | --- |
| `promotion_compensation` | 晋升与加薪 | 晋升标准、薪酬、奖金和资源谈判 | `event_only` |
| `internal_transfer` | 内部转岗 | 留在组织内更换岗位或团队 | `event_only` |
| `management_path` | 管理路径 | 第一次带人和管理方向选择 | `event_only` |
| `layoff_risk` | 裁员与职业风险 | 尚未发生的裁员风险评估和准备 | `event_only` / `routing_only` |
| `resignation_exit` | 离职决策 | 是否离开、如何退出和交接 | `event_only` |
| `side_project_income` | 副业与第二收入 | 副业、利益冲突和时间取舍 | `event_only` |

## 8. 风险与健康主题

| ID | 名称 | 定义 | 默认优先级 |
| --- | --- | --- | --- |
| `toxic_workplace` | 有毒环境 | 持续贬低、威胁、报复、歧视和骚扰 | `routing_only` |
| `burnout_health` | 倦怠与健康 | 无法恢复、身心耗竭和健康受损 | `routing_only` 或 `event_only` |
| `workplace_rights` | 劳动权益 | 合同、社保、工资、解除和法定权益 | `routing_only` |

分类器识别主题和风险，不作法律、医疗或心理诊断。

## 9. 用户意图 intent

| ID | 定义 | 示例 |
| --- | --- | --- |
| `explain` | 理解概念或原因 | “Landing 文档是什么？” |
| `diagnose` | 判断问题性质和主要原因 | “为什么我总接不住任务？” |
| `decide` | 判断要不要做或选哪个 | “要不要参加这次聚餐？” |
| `plan` | 需要多步骤或阶段方案 | “帮我规划入职第一个月。” |
| `next_action` | 只需要眼下第一步 | “我现在先做什么？” |
| `communication_script` | 需要可直接使用的话术 | “怎么跟老板说？” |
| `template` | 需要清单、表格或结构 | “给我一个周报模板。” |
| `prepare` | 为具体事件做完整准备 | “明天第一次 1:1 怎么准备？” |
| `review` | 检查已经完成的内容 | “帮我看看这份周报。” |
| `risk_check` | 判断骗局、违法、安全或严重风险 | “入职前让我交钱正常吗？” |
| `emotional_support` | 当前只有情绪表达，尚无明确行动需求 | “我觉得自己什么都做不好。” |

意图边界：
- “要不要”优先 `decide`；
- 已确定目标问“怎么做”优先 `plan`；
- 只问第一步优先 `next_action`；
- 需要可复制措辞优先 `communication_script`；
- 需要完整事件准备优先 `prepare`；
- 情绪中包含明确选择时，选择意图优先，情绪支持作为辅助。

## 10. 风险等级 risk_level

| ID | 定义 | 处理 |
| --- | --- | --- |
| `normal` | 普通职场问题 | 进入普通流程 |
| `caution` | 持续加班、明显倦怠、反复贬低或较高职业风险 | 增加边界和风险提醒，必要时升级 |
| `high` | 明确骚扰、歧视、报复、违法或严重身心反应 | `ESCALATE` |
| `emergency` | 即时人身安全、自伤或他伤风险 | 立即进入紧急帮助流程 |

多个风险同时出现时选择最高等级，不取平均。

## 11. 普通处理结果 ordinary_decision

| ID | 定义 | 是否检索 SOP | 是否生成任务 |
| --- | --- | --- | --- |
| `EXECUTE` | 用户需要形成行动，且存在值得投入、用户可控的下一步 | 是；允许 `MATCHED` 或 `NO_MATCH` | 用户确认后才生成 |
| `ARCHIVE` | 用户提出了行动或投入判断，但当前不值得继续投入 | 否 | 否 |
| `RESPOND_ONLY` | 用户只需要解释、澄清、情绪承接或简单信息，本轮不需要行动计划 | 否 | 否 |

边界规则：

- `RESPOND_ONLY` 不是“建议做了但没生成任务”，而是本轮本来就不需要任务；
- `ARCHIVE` 必须包含“当前不值得投入”的判断；
- 用户获得回答后主动要求形成行动时，重新进入 `EXECUTE | ARCHIVE` 判断；
- `ESCALATE` 是范围与风险路由，不属于普通处理结果。

## 12. 关键分类边界

### 12.1 求职与待入职

- 未接受 Offer：使用求职主题；
- 已接受 Offer：`career_phase=preboarding`；
- 比较、谈判尚未接受的 Offer：`offer_negotiation`；
- 接受 Offer 后准备第一天：`preboarding`。

### 12.2 入职与具体问题

- 入职早期整体适应：`onboarding_landing`；
- 已有明确协作对象但推进困难：`peer_collaboration`；
- 不知道谁负责、谁评价：`organization_stakeholder`；
- 做不完或先做什么：`workload_priority`；
- 已完成但不可见：`performance_visibility`。

### 12.3 向上沟通与晋升

- 普通目标、反馈、汇报和 1:1：`manager_communication`；
- 明确指向职级、加薪、奖金：`promotion_compensation`；
- 晋升主题为 `event_only`，不得因为用户入职满半年自动召回。

### 12.4 边界、有毒环境与离职

- 普通拒绝、职责争议、抢功、甩锅：`boundary_conflict`；
- 持续人格贬低、威胁、歧视、骚扰或报复：`toxic_workplace`；
- 尚未决定离开、主要处理当前冲突：不自动分类为离职；
- 用户明确比较留下或离开：`resignation_exit`。

### 12.5 工作量与倦怠

- 核心请求是排序、减负和效率：`workload_priority`；
- 核心请求是无法恢复、健康受损：`burnout_health`；
- 严重身体或心理反应提高风险等级。

## 13. 澄清规则

需要澄清：

- 不同答案会改变主主题；
- 不同答案会改变风险；
- 缺失信息可能导致相反判断；
- 用户需要话术但沟通对象或目标不清楚；
- 保存阶段与用户本轮明确状态冲突。

不需要澄清：

- 信息只是锦上添花；
- 可先提供安全、可逆的通用步骤；
- 画像中已有明确且近期的信息；
- 用户只需要概念解释。

每轮最多一个澄清问题。

## 14. 置信度

| 区间 | 含义 |
| --- | --- |
| `0.90-1.00` | 用户明确表达，边界清晰 |
| `0.70-0.89` | 基本明确，存在次要歧义 |
| `0.50-0.69` | 多个分类合理，需要澄清 |
| `<0.50` | 信息严重不足，主题可使用 `unknown` |

职业阶段不能因为低置信度而猜测为其他阶段。
