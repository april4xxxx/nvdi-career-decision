日期：2026-07-20

## 1. 统一结算规则

- 精力、金币、任务、恢复、跨日、心流与成就奖励统一写入 `settlementLedger`。
- 任务使用 `task:<task_id>`，跨日使用 `day:<YYYY-MM-DD>`，成就使用 `achievement:<achievement_id>` 作为幂等键。
- 成就金币与任务金币是同一种金币，全部进入 `gold` 与 `totalGold`；只在结算记录中保留来源说明。
- 成就奖励可以继续推动 50/100/300/500/1000/3000 等累计金币成就；每个成就结算键唯一，因此连锁不会重复发放。
- 称号奖励直接进入 `titles`。
- 新成就日期记录为 `YYYY-MM-DD · 登基第 N 天`。
- 旧存档已解锁但未发奖的成就会补发一次，并写入一条“旧成就奖励补发”起居注。

## 2. 起居注现在记录什么

| 类型 | 当前内容 |
| --- | --- |
| 登基 | 默认“登基第一日”记录 |
| 任务结算 | 任务类别、精力前后值、任务金币、同步解锁的成就、自动到账成就金币/称号、决策来源与知识来源 |
| 恢复结算 | 实际恢复值、恢复前后精力、恢复任务无金币、同步成就奖励 |
| 跨日结算 | 跨越自然日数、实际自然恢复、当前精力、日期类与连续类成就 |
| 成就 | 非任务场景触发的成就名称、显示奖励和自动到账状态 |
| 迁移 | 旧成就奖励补发数量与金币合计 |
| 朱批 | 再议或大胆驳回的决策名称与处理结果 |
| 心流 | 25 分钟专注、精力消耗及无金币说明 |
| 对话归档 | 场景、主题、最近对话摘要、未朱批奏折或未回答追问 |
| 典籍 | 上传书名及其已加入 AI 决策知识的说明 |

任务起居注示例：

```text
任务达成 · 日常 · 结算：精力 -10（100→90）
· 任务金币 +10
· 解锁成就「初入国库、初开窑火……」，成就奖励已自动到账
· 源自决策「……」
```

## 5. 仍需后续观察

- 自动到账允许金币成就形成一次性连锁，第一项任务可能同时触发多项早期奖励；这是当前确认的统一金币规则，需要用真实首周数据判断奖励是否过密。
- 金币消费未上线前，金币是长期累计分数；“量入为出”明确显示为暂未开放。
- 周报/SOP和答辩成就同时支持结构化标签与标题兼容匹配；正式任务模板完善后应优先使用标签。
- 30/60/90日里程碑仍保留，但解锁日期展示已改为真实日期加登基天数；这些门槛可以通过数据表继续调整。
