# 女帝职场决策系统改造记录

## 记录信息

- 日期：2026-07-20
- 版本：1.1.0
- 目标：把纯前端模拟原型改造成可部署、可真实对话、可上传知识、可继续驱动任务与珍宝阁成就的黑客松 Demo。
- 原则：保留现有视觉、业务状态和本地演示能力；新增能力以小范围适配层接入。

## 改造前状态

- 项目为无构建步骤的 HTML/CSS/JavaScript 静态站点。
- `data.brain.analyze()` 通过关键词与固定情景模拟对话、追问和决策。
- `store.js` 已经实现任务投放、完成结算、起居注、金币、精力和 59 项成就互通。
- 藏书阁上传表单只保存书名、作者和摘要，并明确提示“不实际读取文件”。
- 用户状态保存在浏览器 `localStorage(nvdi-full-v1)`。
- 项目目录约 126MB，主要由图片素材构成；尚未初始化 Git 仓库。

## 架构决定

### 1. 保留 Vanilla JS

没有迁移 React、Vue 或 Next.js。现有模块边界清晰，重写框架会增加黑客松风险，却不会直接改善核心体验。

### 2. 同源 Vercel Functions

浏览器只访问 `/api/chat` 和 `/api/knowledge/upload`。OpenAI API Key 存在 Vercel 环境变量中，不进入 HTML、JavaScript bundle 或 GitHub。

### 3. 结构化决策协议

AI 输出被约束为 `dialogue | question | decision`。`decision` 延续原有 `category / mirror / recommend / alt / tasks` 结构，因此可直接进入：

```text
presentDecision
→ applyPizhu
→ deployTasks
→ completeMapTask
→ achievement unlock
→ treasury render
```

服务端还会二次清洗分类、数值、文本长度和任务数量，避免异常模型输出破坏前端状态。

### 4. 两层知识库

- 公共知识库：维护在 `knowledge/`，由脚本预先上传，使用 `OPENAI_VECTOR_STORE_ID`。
- 用户知识库：每个浏览器首次上传时建立单独 Vector Store，后续继续复用。

服务端不直接信任客户端提交的 Vector Store ID，而是返回 HMAC 签名令牌。这样一个用户不能通过修改请求任意检索同一 OpenAI 项目下的其他知识库。

### 5. 可用性优先

真实 API 因未配置、超时、限流或上游错误不可用时，前端自动回退原有 `data.brain.analyze()`。这保证现场网络波动不会让 Demo 失去主流程。

### 6. 状态范围

黑客松版本继续使用 localStorage 保存进度。它是真实、可交互的用户数据，但只存在当前浏览器。跨设备同步需要认证与数据库，留到下一阶段。

## 数据流记录

### AI 对话

1. 用户输入进入 `conversation.js`。
2. 前端采集有限状态：精力、金币、场景、未完成任务、近期起居注和藏书标题。
3. `js/api.js` POST 到 `/api/chat`。
4. 服务端验证大小和频率，验证知识库签名令牌。
5. Responses API 使用公共及用户 Vector Store 检索，并按 JSON Schema 返回结果。
6. 服务端清洗结果，前端渲染对话、追问或奏折。
7. 用户朱批同意后继续走原有任务和成就引擎。

### 典籍上传

1. 用户选择 TXT、Markdown、PDF，或把文本心得生成 TXT。
2. 浏览器上传到 `/api/knowledge/upload`。
3. 服务端新建或复用私有 Vector Store，上传 OpenAI File 并等待索引。
4. 服务端返回签名知识库令牌。
5. `store.js` 把令牌和书籍元数据保存到 localStorage。
6. 后续 `/api/chat` 验证令牌后启用该知识库。

## 安全与成本控制

- API Key 不进入前端或仓库。
- 聊天消息最长 2000 字，历史最多 12 条，输出上限 2600 tokens。
- 单服务实例默认每 10 分钟最多 24 次对话、8 次上传。
- 上传文件最大 4MB，扩展名白名单为 `.txt`、`.md`、`.pdf`。此限制为 Vercel 4.5MB Function 请求体上限预留 multipart 开销。
- 聊天和上传函数显式配置为最长 30 秒，覆盖 AI 推理和短时间索引等待。
- 模型通过 `OPENAI_MODEL` 可配置；2026-07-20 根据 OpenAI 官方最新模型解析结果，将默认值校正为 `gpt-5.6-sol`，reasoning effort 为 low。
- 正式公开前仍建议加入持久限流、验证码/登录，以及 OpenAI Project 预算限制。
