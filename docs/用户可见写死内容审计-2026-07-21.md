# 用户可见写死内容审计

## 审计边界

本次只审计会进入用户任务、奏折或决策结果的写死内容。视觉文案、角色开场白和明确由用户启动的演示素材不属于自动任务数据。

## 已移出普通流程

| 内容 | 旧行为 | 处理 |
| --- | --- | --- |
| 结业答辩、本周周报、喝咖啡认识前辈 | 入职完成或空存档时自动进入地图 | 停止播种；v8 首次刷新清理旧数据及近似改写 |
| `SCENARIOS` 情景决策 | AI 失败后静默接管并可生成真实任务 | 仅在用户主动点击“演示”时启用 |
| 旧演示遗留任务 | 旧版本模板曾混入真实地图 | v8 仅清理旧模板；新演示生成且原本不存在的任务会保留，并参与后续语义查重 |
| `genericDecision` 第一小步 | 追问后响应异常时强制生成任务 | 不再强制落任务，改为展示真实响应或明确失败 |
| 预言模式行业分享 | 永远读取第一条固定情景 | 改为读取当前真实待批奏折；没有待批则显示空态 |

## 保留的明确演示或产品内容

| 内容 | 保留原因 | 隔离方式 |
| --- | --- | --- |
| 左下角演示的行业分享、课程、疲惫情景 | 用户明确启动，用于展示产品能力 | `App.demo.isRunning()` 为真时才允许硬编码情景运行 |
| 钦天监每日微探索 | 产品定义的恢复机制，不冒充用户职场任务 | 独立 `daily-mystic` 来源、无金币、固定在钦天监 |
| 宫廷场景开场白和三位大臣固定招呼 | 角色与界面文案，不写入任务池 | 只显示为对话 UI 文案 |
| 成就目标和默认藏书 | 产品规则与知识展示 | 不自动生成职场待办 |
| 五个任务场景的“任务范例”空态卡 | 让地图不会因没有真实待办而显得空 | 不写入 `mapTasks`，只展开 AI 议事 |

## 防回归规则

- 普通 AI 失败不得调用硬编码决策生成任务。
- 新用户完成入职后任务池允许为空。
- 旧三条模板、常见改写和旧演示遗留任务只在 v8 迁移时清理一次。
- 前端不得通过任务标题关键词篡改分类；真实任务的场景投放只读取后台 AI 返回的 `cat`。
- 场景任务范例不得进入真实任务池、对话的 `pendingTasks`或办结结算。
- 同一未完成任务不能通过更换分类或场景绕过去重。
