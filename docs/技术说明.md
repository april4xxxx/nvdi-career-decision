# 女皇入朝：开发、部署与系统说明

本文档收录项目能力、运行方式、知识库、数值规则、API、安全边界与部署信息。项目介绍与在线体验请返回 [README](../README.md)。

## 当前能力

- 真实 DeepSeek-V4-Pro 对话，API Key 仅由 Vercel 服务端读取。
- 服务端返回结构化的对话、追问或决策奏折；AI 只估算时长，程序按固定档位计算精力与金币。
- AI 任务进入原有地图任务系统；完成后自动结算精力、金币并解锁珍宝阁成就。
- 新奏折投放任务时会检查同场景的未完成任务；标题高度重合的任务会关联到原任务，不再生成重复待办，已完成任务仍可在之后重新创建。
- 当天对话按宫廷场景分别保存；切换地图后返回可继续查看消息、追问、待批奏折和未发送草稿，跨日后旧对话自动归入起居注。
- 起居注只归档实际商讨内容，会自动过滤切换大臣提示、固定开场白和重复消息；标题避免重复场景名，多轮商讨按行展示。
- 对话明确区分“收起”和“新对话”：新会话以“朝堂议事 / 御花园议事”等当前场景命名，收起后可一键继续；存在草稿、追问或待批奏折时会先提示确认。
- 宫廷地图只标记需要处理的“待答 / 待批”状态，普通对话不额外占用侧栏空间；重复任务也会在朱批前提示将关联原待办。
- 左侧宫廷地图会记住当前滚动位置；切换场景或刷新对话状态后，不会自动跳回侧栏顶部。
- 任务投递与自动演示的场景预览不计为到访；只有用户点击地图或“前往场景”才推进探索类成就。
- 公共《女帝职场决策原则》自动参与每次对话。
- 已接入版本化 SOP 运行时发布管道；只有库级与模块级均为 `ACTIVE` 的内容才会进入线上检索。
- 支持 TXT、Markdown、PDF 真实文字提取；浏览器结合问题相关度与入职天数，从用户典籍中选择相关片段提供给 AI。
- 用户上传文本只保存在自己的浏览器，服务端不建立长期文件存储。
- API 不可用时自动切回本地情景大脑，保证演示不中断。

## 本地预览

纯视觉与本地演示模式可直接使用静态服务器：

```bash
python3 -m http.server 4173
```

然后访问 `http://localhost:4173`。静态服务器不提供 `/api`，因此对话会自动使用本地演示大脑。

测试真实 API 时，项目需先与 Vercel 项目绑定并拉取本地环境变量：

```bash
npm install
vercel link
vercel env pull .env.local --environment production
npm run local -- --listen 127.0.0.1:3000
```

然后访问 `http://127.0.0.1:3000`。修改前端文件后刷新浏览器即可；修改 `/api` 后 Vercel 开发服务器会重新加载函数。不要把 `.env.local` 提交到 GitHub。

不要把 `vercel dev` 配成 `package.json` 的 `dev` 脚本：Vercel 会把它识别为 Development Command 并再次调用，形成递归。本项目使用名称不同的 `local` 脚本规避该问题。

## 环境变量

| 名称 | 是否必填 | 用途 |
| --- | --- | --- |
| `DEEPSEEK_API_KEY` | 真实 AI 必填 | DeepSeek Key，仅由服务端读取 |
| `deepseek` | 兼容名称 | 当前 Vercel 已使用此名称；代码会自动识别 |
| `DEEPSEEK_MODEL` | 可选 | 默认 `deepseek-v4-pro` |
| `CHAT_RATE_LIMIT` | 可选 | 单服务实例每 10 分钟对话请求上限，默认 24 |

历史部署中的 `APP_SESSION_SECRET` 只服务于已移除的 OpenAI Vector Store 方案，当前版本不再读取，可在确认 1.2.0 稳定后从 Vercel 删除。

## 知识库工作方式

1. 公共知识原稿维护在 `knowledge/女帝职场决策原则.md`，核心规则随每次对话发送给 DeepSeek。
2. 用户在藏书阁上传 TXT、Markdown 或带可选文字的 PDF，最大 4MB。
3. `/api/knowledge/upload` 提取最多 60,000 字并立即返回，不把原文件写入数据库或对象存储。
4. 浏览器把提取结果保存在 `localStorage(nvdi-full-v1)`。
5. 发起对话时，浏览器以当前问题做轻量词法检索，并根据当前入职阶段（1-7、8-30、31-60、61-90 天等）对匹配片段加权，最多选择 4 个相关片段、18,000 字发送到服务端。
6. DeepSeek 使用了典籍时，会在奏折 `sources` 中注明典籍名。

### 正式 SOP 的发布方式

项目内 `sop/04-职场新人SOP库.json` 的 `REVIEWED` 只表示内容已经审阅，不代表已经上线。发布包生成器只收录模块级 `ACTIVE`，并要求库级状态也为 `ACTIVE`：

```bash
node scripts/build-sop-runtime.mjs \
  --library 'sop/04-职场新人SOP库.json' \
  --taxonomy sop/data/career-taxonomy.v2.2.json \
  --output api/_generated/sop-runtime.js
```

生成结果由 `/api/chat` 自动读取；先按入职阶段硬过滤，再执行排除条件门和相关性排序，最多提供 3 个候选 SOP。AI 实际采用模块时，任务会记录 `SOP:<sop_id>｜<title>` 作为参考来源。

当前项目主库完整保存 22 个 `REVIEWED` 模块。首批四个低风险模块已通过历史真实 DEV 的 16 个场景及项目内结构、检索和回归测试；新的 HOLDOUT/SAFETY 已完成结构 dry-run，但系统级租户策略禁止向外部 DeepSeek 传输项目数据，因此尚未完成模型门，也未激活。生成包继续保持 `BLOCKED_NO_ACTIVE_MODULES`、模块数为 0。详细门禁见 `sop/10-首批ACTIVE模块发布记录.json`。

## 任务经济规则

| 预计时长 | 普通任务档位 | 耗精力 | 完成奖励 |
| --- | --- | ---: | ---: |
| 5-30 分钟 | LIGHT | 10 | 10 金 |
| 31-90 分钟 | MEDIUM | 20 | 20 金 |
| 91-240 分钟 | HEAVY | 30 | 30 金 |

恢复采用独立档位：10-15 分钟恢复 10，30-45 分钟恢复 20，60-90 分钟恢复 30。

普通任务金币仅在用户确认任务完成时发放；成就金币在解锁时自动到账。恢复任务、方案生成、朱批同意、典籍上传、归档和进入心流本身不产生金币；模型返回的自定义数值会被服务端忽略。精力上限为 150，每个真实跨日自然恢复 30，重复刷新不会重复结算。

成就达成后，展示的金币或称号会立即自动到账，不设置待领取步骤。任务金币与成就金币使用同一个余额和历史累计；所有资源变化写入带幂等 ID 的结算账本，并在起居注中显示本次精力、金币和成就变化。

扫描图片型 PDF 没有可提取文字，当前会提示先进行 OCR 或转换为 TXT；这是黑客松版本的明确边界。

## Vercel 部署

当前 Vercel 项目：`april4xxxxs-projects/nvdi-career-decision`，生产域名为 `nvdi-career-decision.vercel.app`。

1. Framework Preset 选择 **Other**，项目根目录保持仓库根目录。
2. 在 Vercel Project Settings → Environment Variables 添加 `DEEPSEEK_API_KEY`；现有部署也兼容变量名 `deepseek`。
3. 每次修改环境变量后重新生产部署。
4. 依次验证：Onboarding、真实对话、追问、奏折、朱批同意、完成任务、珍宝阁解锁、典籍上传与典籍引用。

## API 约定

### `POST /api/chat`

浏览器发送当前消息、大臣、最近对话、有限应用状态和经过本地筛选的知识片段。服务端调用 DeepSeek Chat Completions，并返回：

```json
{
  "result": {
    "type": "dialogue | question | decision",
    "topic": "主题",
    "message": "普通回复",
    "question": null,
    "decision": null
  },
  "meta": {
    "mode": "deepseek",
    "model": "deepseek-v4-pro",
    "usedKnowledgeBase": true,
    "userKnowledgeDocuments": 1
  }
}
```

### `POST /api/knowledge/upload`

接收 `multipart/form-data` 中的 `file` 和可选 `title`，返回文件元数据与提取文本。服务端不需要接触 DeepSeek Key，也不长期保存文件。

## 安全与成本边界

- DeepSeek API Key 不下发浏览器、不写进仓库。
- 对话限制请求大小、消息长度、历史条数、输出 token 和请求频率。
- 用户文件限制为 TXT、Markdown、PDF，最大 4MB；提取文本最多 60,000 字。
- 每轮最多发送 18,000 字用户知识，避免无边界消耗上下文和费用。
- 用户知识存在浏览器 localStorage；清除网站数据会清除这些典籍。
- 当前内存限流只适合黑客松 Demo。公开流量增大时应换成持久限流，并增加验证码或登录。
- SOP 发布遵守 `DRAFT → REVIEWED → ACTIVE`；生成器不提供跳过评测或强制包含 `REVIEWED` 的开关。
