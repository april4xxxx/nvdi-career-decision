# Changelog

## Unreleased

### Changed

- 对外品牌名称统一为“女皇入朝｜AI 职场心流决策地图”；应用顶栏使用短名“女皇入朝”，GitHub 仓库名和 Vercel 固定域名保持不变。

### Fixed

- 完成用户可见写死内容审计：取消自动播种“结业答辩 / 本周周报 / 喝咖啡认识前辈”，v8 刷新时清理旧存档、改写副本及旧演示遗留任务；普通 AI 故障不再切入硬编码决策或落任务，硬编码情景仅在用户主动演示时启用且结束后自动清场。
- 任务分类新增语义兜底：“约咖啡 / 认识前辈”归探索、“休息 / 离屏 / 恢复精力”归钦天监；完全相同的在办任务即使被 AI 错分到不同场景也会拦截合并。
- AI 不再把“已有待办”改写成本轮新任务；任务查重会忽略“完成 / 准备 / 的”等计划措辞差异，旧存档中类似“完成结业答辩 / 准备结业答辩大纲”的在办重复卡刷新后自动合并。
- AI 大臣不再误用女帝自称“朕”：系统提示锁定君臣身份，服务端与前端双重纠正大臣消息，旧会话刷新后同步修复；用户自己的“朕”保持不变。
- AI 任务标题不再暴露 `[main] / [daily] / [explore] / [delay] / [mystic]` 内部分类标记；服务端响应、前端任务落库和旧存档加载三层同时清理，已有任务及对话公告刷新后自动恢复为用户可读标题。
- 左下角“演示”切换为宁静观看节奏：场景跳转留出 1.8s，普通窗口稳定 1.2s，关键内容阅读 2.4–3.2s，操作前高亮 1s，逐字输入由 34ms/字放缓至 54ms/字。
- 所有 AI 生成回合统一显示大臣三点思考气泡，即使本地回退或极快响应也保留至少 420ms 的可感知反馈；请求完成或失败后自动收起，同时最多只存在一个。
- 收起态恢复最初的“大臣一句话＋常驻输入框＋探讨国事”结构，移除对话框内重复的“继续对话 / 新对话”按钮；新对话入口仅在展开顶栏保留。
- 侧栏展开态的 `<` 与收起态的 `>` 改用同一水平基准，到各自侧栏右边缘的留白保持一致。
- 对话展开顶栏与收起摘要统一为同一高度和按钮尺寸，收起态不再显得过于厚重。
- 六部等任务较多的场景在浏览器放大后，任务区改为内部滚动，底部对话框与输入区不再被挤出或裁掉；低高度下同步压缩次要场景信息和奏折匣入口。
- 起居注归档不再写入“切换大臣”系统提示、固定开场白和重复消息；没有实质商讨时不生成空归档，旧存档中的同类异常条目会自动清理。默认标题不再重复成“御花园·御花园议事”，多轮商讨改为换行展示，避免 `。；` 连写。
- 左侧宫廷地图在切换场景、恢复场景对话或刷新状态后保持原滚动位置，不再反复跳回侧栏顶部。
- 宫廷地图移除重复的“对话”标签，仅保留“待答 / 待批”提醒；侧栏折叠箭头改为单箭头，并与窄栏头像分行排列。
- 新会话标题恢复为“当前场景名＋议事”，与宫廷语境保持一致。
- 任务办结弹窗改为窄版“任务奏报”样式，重新梳理任务标题、精力/金币结算与操作层级，减少过量留白。
- 对话右上角的模糊“×”改为明确的“收起”；待批奏折收起后，下方主按钮显示“继续批阅”，不会在奏折不可见时误触批准。
- 新对话会把上一段商讨归入起居注，并只重置当前场景；未发送草稿、未答追问或待批奏折存在时先显示风险确认。
- 宫廷地图新增“对话 / 待答 / 待批”状态标记，帮助用户找回分散在不同场景的进行中商讨。
- 奏折在朱批前即提示与现有待办的重合关系，并提供查看原任务入口。
- 新奏折与同场景未完成任务高度重合时，改为关联原任务及其来源，不再产生两个重复待办；已完成任务不阻止后续重新创建。
- 对话会按当天与宫廷场景保存消息、追问、待批奏折、选择路径和输入草稿；切换地图后返回原场景可继续对话，跨日时自动归档到起居注。
- 待批奏折下，只有输入为空时点击「同意」才会批准；已填写的再议补充内容会发给 AI 重新拟策。
- 普通 `dialogue` 闲聊不再消耗唯一一轮决策追问机会，只有真正的 `question` 会更新追问状态。
- 用户典籍检索不再发送总分为 0 或负分的无关片段，同时保留入职阶段相关性加权。
- AI 只估算任务时长；普通与恢复任务统一按 LIGHT/MEDIUM/HEAVY 固定档位计算精力和金币，旧任务在加载时迁移到同一规则。
- 只有确认完成任务才发放金币；朱批同意、典籍上传和心流计时不再额外发币。
- 新增正式 SOP 发布生成器和服务端检索入口；只允许 `ACTIVE` 模块进入生成包，并在模型调用前按阶段和排除条件确定性过滤。
- 为两项 DEV 严格错误增加回归门：权限未开通时禁止工具流程，尚无成果时禁止成果记录。

### Tests

## 1.2.0 - 2026-07-20

### Changed

- 真实对话提供方由 OpenAI Responses API 改为 DeepSeek Chat Completions。
- 默认模型固定为官方模型 ID `deepseek-v4-pro`，并兼容 Vercel 现有变量名 `deepseek`。
- 公共决策原则改为每轮直接注入上下文，不再依赖外部 Vector Store。
- 用户典籍改为服务端真实提取文字、浏览器本地保存和对话前轻量相关片段检索。
- DeepSeek 的 JSON 输出继续经过原有奏折结构清洗，任务、金币、精力和成就链路保持不变。

### Added

- 新增 TXT、Markdown 和可选文字 PDF 的提取结果、字符数与截断状态。
- 新增 PDF 提取依赖 `pdf-parse@^2.4.5`、Vercel Node 图形兼容依赖 `@napi-rs/canvas@0.1.80` 和依赖锁文件。
- Vercel Function 显式包含 `pdf.worker.mjs`，避免 Node File Trace 遗漏 PDF.js worker。

### Removed

- 移除 OpenAI Files、Vector Store、File Search、知识库令牌签名和公共知识库播种脚本。

### Security

- 用户原文件不长期保存在服务端；提取文本仅返回并保存在当前浏览器。
- 每轮用户知识上下文限制为最多 6 份输入、18,000 字，前端实际最多选择 4 个相关片段。

## 1.1.0 - 2026-07-20

### Added

- 新增 Vercel `POST /api/chat`，通过 OpenAI Responses API 返回结构化职场决策。
- 新增 Vercel `POST /api/knowledge/upload`，支持 TXT、Markdown、PDF 进入用户私有知识库。
- 新增公共知识库初始化脚本和首份决策原则知识文件。
- 新增前端 API 适配器、知识库来源展示、上传状态和 AI 不可用回退提示。
- 新增环境变量模板、部署说明、安全说明和 Node 自动测试。

### Changed

- 对话从固定情景库优先切换为真实 AI；API 不可用时仍使用原情景库。
- 藏书阁“上传典籍”由纯演示表单改为真实文件上传与索引。
- 状态仓库新增签名知识库令牌，保持上传典籍与后续 AI 对话互通。
- 根据 2026-07-20 OpenAI 官方模型解析结果，将默认 API 模型校正为 `gpt-5.6-sol`。

### Security

- API Key 只由 Vercel 服务端读取。
- 对话加入输入裁剪、输出约束和基础限流。
- 用户 Vector Store ID 经 HMAC 签名后才允许在对话检索中使用。
- 上传限制为 4MB，并为聊天与知识索引函数配置 30 秒执行时间以适配 Vercel Hobby 运行约束。
